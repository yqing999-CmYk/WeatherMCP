# WeatherMCP

A collection of **Model Context Protocol (MCP) servers** built in Python. The project exposes real-time US weather data and local host system information as MCP tools that any MCP-compatible AI client (e.g. Claude Desktop, Cursor, VS Code) can call directly.

---

## Introduction

WeatherMCP demonstrates how to build MCP servers that give AI assistants access to live, external data sources. Three separate servers are included:

| Server | File | Purpose |
|--------|------|---------|
| `weather` | `weather.py` | Fetch US weather alerts and forecasts via the NOAA NWS API |
| `new_weather_mcp` | `newWeather.py` | Alternate weather server (v2 user-agent) |

A diagnostic tool (`mcp_logger.py`) is also included to transparently log the raw MCP JSON-RPC traffic between a client and any server.

## How It Works

### MCP Protocol

All servers communicate over **stdio** using the JSON-RPC 2.0-based MCP protocol. An MCP client (e.g. Claude Desktop) launches the server as a child process and exchanges JSON messages through stdin/stdout.

```
AI Client  ──(JSON-RPC over stdio)──►  MCP Server  ──►  External API / OS
```

### Weather Server (`weather.py`)

Registers two tools with FastMCP:

- **`get_alerts(state)`** — queries `https://api.weather.gov/alerts/active/area/{state}` and returns formatted active weather alerts for a US state (two-letter code, e.g. `CA`).
- **`get_forecast(latitude, longitude)`** — calls the NWS `/points` endpoint to resolve a grid, then fetches the 5-period forecast for those coordinates.

All HTTP calls are made asynchronously with `httpx.AsyncClient`.

### MCP Logger (`mcp_logger.py`)

A transparent stdio proxy for debugging. It wraps any MCP server command, forwards stdin/stdout/stderr unchanged, and writes every message to `mcp_io.log` with `input:` (input) and `output:` (output) prefixes.

```
AI Client ──► mcp_logger.py ──► actual MCP server
                   │
                   └──► mcp_io.log
```

## How to Run

### Prerequisites

- Python 3.11 installed (`py -3.11 --version` to verify)
- `uv` package manager ([install guide](https://docs.astral.sh/uv/getting-started/installation/))

### Install dependencies

```bash
# From the project root
uv sync
```

### Run the weather server directly

```bash
uv run python weather.py
```

```

### Run via the package entry point

```bash
uv run weathermcp
```

### Use with Claude Desktop

Add a server entry to your Claude Desktop config file
(`%APPDATA%\Claude\claude_desktop_config.json` on Windows):

```json
{
  "mcpServers": {
    "weather": {
      "command": "uv",
      "args": [
        "--directory",
        "C:/Users/yqing/WeatherMCP/weathermcp",
        "run",
        "python",
        "weather.py"
      ]
    }    
  }
}
```

### Enable MCP I/O logging (optional)

Wrap any server command with `mcp_logger.py` to capture raw traffic:

```json
{
  "mcpServers": {
    "weather-debug": {
      "command": "uv",
      "args": [
        "--directory", "C:/Users/yqing/WeatherMCP/weathermcp",
        "run", "python", "mcp_logger.py",
        "uv", "run", "python", "weather.py"
      ]
    }
  }
}
```

Logs will appear in `mcp_io.log` in the project root.

## Deployment

### Packaging as a distributable wheel

Use `uv` to build a wheel:

```bash
uv build
```

This creates a wheel and sdist in `dist/`. The wheel can be installed anywhere Python 3.11+ is available:

```bash
pip install dist/weathermcp-0.1.1-py3-none-any.whl
weathermcp   # runs the entry point
```

### Running as a systemd service (Linux)

```ini
# /etc/systemd/system/weathermcp.service
[Unit]
Description=WeatherMCP Server
After=network.target

[Service]
ExecStart=/path/to/.venv/bin/python /path/to/weather.py
Restart=on-failure
StandardInput=null

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl enable --now weathermcp
```

## Docker

### Dockerfile

```dockerfile
FROM python:3.11-slim

# Install uv
RUN pip install uv

WORKDIR /app

# Copy dependency files first for layer caching
COPY pyproject.toml uv.lock ./

# Install dependencies into the system Python (no venv needed in container)
RUN uv sync --no-dev --system

# Copy source code
COPY *.py ./

# Default: run the weather server
CMD ["python", "weather.py"]
```

### Build and run

```bash
# Build the image
docker build -t weathermcp .

# Run the weather server (stdio mode — attach stdin for MCP client)
docker run -i weathermcp


### Docker Compose

```yaml
# docker-compose.yml
services:
  weather:
    build: .
    stdin_open: true
    tty: false
    command: python weather.py

  hostinfo:
    build: .
    stdin_open: true
    tty: false
    command: python hostInfo_server.py
    # Give psutil access to real host system metrics
    pid: host
    volumes:
      - /proc:/proc:ro
      - /sys:/sys:ro
```

```bash
docker compose up
```

> **Note:** The host info server uses `psutil` to read system metrics. Inside Docker it will report container-level stats unless you mount `/proc` and set `pid: host` as shown above.
